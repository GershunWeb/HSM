# Развертывание Healthy Mind Space (статический сайт + WebRTC сигналинг)

Короткая инструкция для бесплатного размещения статического сайта и работы видеозвонков между устройствами.

1) Хостинг статического сайта (бесплатно)

- GitHub Pages (рекомендуется): создайте репозиторий, запушьте содержимое папки проекта (включая `tests/`, `doctor.html`, `imdex.html`), в Settings → Pages выберите ветку `main`/`gh-pages` и папку `/` или `/docs`. GitHub Pages предоставляет HTTPS автоматически.

- Netlify / Vercel: обе платформы поддерживают drag-and-drop (дропните папку) или деплой из GitHub. Тоже дают HTTPS.

2) Сигналинг для WebRTC (чтобы звонки работали между разными устройствами)

На локальной версии сайт использует простой `localStorage`-сигналинг, который работает только между вкладками одного браузера. Для звонков между разными устройствами нужно использовать общий signaling-сервер.

Вариант без сервера (бесплатно): Firebase Realtime Database

- Создайте проект в https://console.firebase.google.com/
- В разделе Realtime Database создайте базу данных и выберите режим "test" (для тестирования). Для продакшена настройте правила безопасности.
- Скопируйте конфигурацию проекта и создайте в корне сайта файл `firebase-config.js` со структурой из `firebase-config.example.js`.
- После добавления конфигурации страница `doctor.html` автоматически подключит Firebase и будет использовать Realtime Database для обмена offer/answer/ICE.

Пример структуры в базе (используется автоматически):

signals/{doctorId}/offer -> {from, desc}
signals/{doctorId}/answer -> {from, desc}
signals/{doctorId}/ice/{pushId} -> {from, cand}

3) Как задеплоить

- Инициализируйте git, закоммитьте проект и запушьте в GitHub.
- Включите GitHub Pages в настройках репозитория или подключите Netlify/Vercel.
- Создайте `firebase-config.js` в корне (не коммитите секреты в публичный репозиторий — используйте environment variables для приватных настроек в продакшне).

4) Тестирование

- Откройте сайт по HTTPS: `https://<your-user>.github.io/<repo>/doctor.html?id=1`
- Откройте страницу на другом устройстве и попробуйте позвонить — теперь сигналинг идёт через Firebase, звонки должны работать между устройствами.

5) Ограничения и безопасность

- На этапе тестирования можно использовать правила Realtime Database с открытым доступом, но для реального сервиса обязательно настройте аутентификацию и более строгие правила.
- Для продакшена используйте проверку прав пользователя, сервер для записи брони и истории чатов, и безопасное хранение секретов.
